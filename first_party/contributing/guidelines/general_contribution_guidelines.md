# General Contribution Guidelines

## Core Principles

### 1. Collaboration
Code is a shared resource. We optimize for the team, not the individual.
- Engage in discussions during code reviews.
- Be open to feedback and alternative approaches.

### 2. Respect
Treat every contributor with respect.
- Critique the code, not the person.
- Assume positive intent.

### 3. Iteration
Perfection is the enemy of progress.
- Submit small, atomic changes.
- It is better to land a "good enough" improvement today than a "perfect" one next week.

### 4. Ownership
Leave the codebase better than you found it.
- Fix small typos or bugs when you see them (scope permitting).
- Update documentation if it is out of date.
