# TODO Standard

The standard for TODO comments in this repository.

## Scope

All TODOs in this repository must conform to this standard; however, the contents of
[third_party](/third_party) are exempt as they originate from other repositories.

## Formatting

TODOs must be associated with a GitHub issue using the format `TODO(#$issue_number): $message`,
for example `TODO(#123): Write the implementation.`. This ensures all incomplete work is tracked in
a centralised database for auditing and organisation.
