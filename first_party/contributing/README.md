# Contributing

This package contains directives for contributing to this repository.

## Context

Contributing to a shared space requires alignment between contributors to prevent fragmentation and
reduce unnecessary work; however, designing and implementing such agreements is a difficult task
given the inherent subjectivity and complexity of human existence. A rule which is clear to one
party may be vague to another, context can change everything, and employing an objective third party
is not a scalable solution, even with artificial intelligence.

The first attempt to solve this problem culminated in the creation of the directives package which
contained rules for documentation and code, but over time, a critical issue arose: while some
details can be objectively verified and enforced, such as the use of a particular list style in
README documents, there are other requirments that inherently involve a degree of subjectivity. A
perfect example is the tone and style used in documentation. In an attempt to maintain a standard
for contributions, particularly documentation generated by artificial intelligence, the directives
were updated to include the requirements that documentation be objective (amongst other requirments)
but this also created issues. Objectivity itself is difficult to evaluate, but more importantly,
rigid and unyielding adherence to rules stifled the very creativity and freedom of expression
required to produce documentation. In order to find a balance between rigid rules general best
practices, and subjective experience, the directives were moved into this package, and the
directives were organized by component rather than type. Each component may contain a mix of
standards (absolute rules), practices (established conventions), and guidelines (subjective advice).

## Definitions

Throughout the documents in this package, the following terms are used with specific meaning:

1. Contributor: A person who makes contributions to this repository (i.e. maintainers and
   reviewers).
1. Consumer: A person who consumes the content of the repository directly (i.e. API users).
1. Uses: A person or organisation which consumes the artefacts produced by the repository (i.e. app
   users).
1. Directive: A standard, practice, or guideline.

## Components

The directives in this package are organized into the following components:

- [Documentation](documentation)
- [Code Blocks](code_blocks)
- [Tooling](tooling)
- [Repository](repository)

This component-based architecture ensures all directives for a topic are colocated. A previous
iteration divided them into layers (i.e. standards package, guidelines package, and practices
package) but this caused related material to be distributed across multiple layers, thereby
hindering comprehension.
