# Contributing

Standards and guidelines for contributing to this repository.

## Context

Contributing to a shared space requires alignment between contributors to prevent fragmentation and
reduce unnecessary work; however, designing and implementing such agreements is a difficult task
given the inherent subjectivity and complexity of human existence. A rule which is clear to one party may be vague
to another, context can change everything, and employing an objective third party is not a scalable
solution, even with artificial intelligence.

The first attempt to solve this problem culminated in the creation of the [standards] package which
contained rules for documentation and code, but over time, a critical issue arose: while some
details can be objectively verified and enforced, such as the use of a particular list style in
README documents, there are other requirments that inherently involve a degree of subjectivity. A
perfect example is the tone and style used in documentation. In an attempt to maintain a standard
for contributions, particularly documentation generated by artificial intelligence, the standards
were updated to include the requirements that documentation be objective (amongst other requirments)
but this also created issues. Objectivity itself is difficult to evaluate, but more importantly,
rigid and unyielding adherence to rules stifled the very creativity and freedom of expression
required to produce documentation. In order to find a balance between rigid rules
general best practices, and subjective experience, the standards package was moved into this
package, and two accompanying directories were added: guidelines, and practices packages. Together
they set expectations that must be followed (standards), help contributors express themselves with
enough structure to maintain repository-wide cohesion (guidelines), and document the established 
processes and best practices (practices). 

## Contents

The contents of this package are divided between three subpackages:

1. [Guidelines](guidelines), which are general best practices, and contain the subjective aspects that are hard
to enforce but easy to describe.
2. [Standards](standards), which are absolute rules that must be followed, and can be verified by an algorithm.
3. [Practices](practices), which are records of the pre-existing systems and strategies used across the
repository.

The general rule of thumb for categorisation is: If the process should be followed because it's the status quo and/or maintains convention, it's a

## Definitions

Throughout the documents in this package, the following terms are used with specific meaning:

1. Contributor: A person who makes contributions to this repository (i.e. maintainers and reviewers).
1. Consumer: A person who consumes the content of the repository directly (i.e. API users).
1. Uses: A person or organisation which consumes the artefacts produced by the repository (i.e.
app users).
1. Directives: The collective set of standards, practices, and guidelines.

## Scope

The directives in this package apply to the entire repository, except for the third party package,
which is explicitly exempt since its contents are derived from external
sources.
